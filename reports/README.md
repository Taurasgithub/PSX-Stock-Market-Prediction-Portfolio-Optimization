# Final Research Outputs

Generated by `scripts/generate_research_outputs.py`, which only calls
existing functions in `src/` (no new algorithms, no changes to any
model).

## What's included in this delivery (generated now, real data)

- `tables/portfolio_metrics.csv` / `.md` — Sharpe Ratio & Maximum
  Drawdown (plus annual return/volatility) for Equal Weight,
  Mean-Variance, and Max Sharpe strategies, from
  `src.portfolio_optimization.rolling_backtest` on the real KSE-100
  daily price history in `Data/`.
- `figures/efficient_frontier.png` — Monte Carlo random-weight
  portfolios (risk vs. return, colored by Sharpe ratio) with the
  Equal-Weight / Mean-Variance / Max-Sharpe portfolios overlaid,
  computed with `src.portfolio_optimization`'s actual optimizer
  functions.
- `figures/cumulative_returns.png` — cumulative growth of $1 for each
  strategy's backtested returns.

**Caveat:** the Mean-Variance and Max Sharpe strategies need an
expected-return input. Since generating that input for real requires
training the project's GRU/BiLSTM models (see below), these two
outputs currently use a naive trailing rolling-mean of realized
returns as a placeholder — this is flagged in the table's `.attrs["note"]`
and in each figure's subtitle. It is **not** a new forecasting
algorithm, only a stand-in input value. Re-run
`run_portfolio_tier(predicted_returns=...)` with the trained model's
real forecasts to replace it.

## What requires the full modeling environment

This sandbox does not have network access and could not install
`tensorflow`, `xgboost`, `statsmodels`, `shap`, or `keras-tuner` (see
`requirements.txt`). The following are fully implemented and ready to
run, but were **not executed**, and no numbers/plots for them are
included, to avoid fabricating results:

- `tables/forecasting_metrics.csv` / `.md` — RMSE/MAPE for ARIMA,
  XGBoost, Univariate LSTM, Multivariate LSTM, GRU, and BiLSTM, plus
  Diebold-Mariano test p-values for each deep learning model against
  both baselines (`src/statistical_tests.py`).
- `figures/gru_shap_summary.png` — SHAP summary plot for the trained
  GRU model (`src/explainability.py`).
- `figures/training_loss_curves.png` — train vs. validation loss per
  epoch for the Multivariate LSTM, GRU, and BiLSTM models.

### To generate these

```bash
pip install -r requirements.txt
python scripts/generate_research_outputs.py
```

This runs both the portfolio tier (above) and the modeling tier,
producing all six outputs together, with the Mean-Variance/Max Sharpe
strategies automatically using the trained GRU/BiLSTM forecasts
instead of the naive placeholder once the modeling tier's predictions
are wired into `run_portfolio_tier(predicted_returns=...)`.
