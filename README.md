# Stock Returns Prediction & Portfolio Optimization
A research thesis entailing the prediction of stock returns using Long Short Term Memory (LSTM) neural network designs and portfolio optimization. Specifically, this study compares the prediction performance of a univariate and multivariate LSTM after which the return predictions from both models are fed into a mean variance optimizer and the final portfolios constructed are evaluated.

## Problem Statement
Classical portfolio optimization techniques often use historical prices and returns as model inputs which has limited influence on future stock behaviour therefore obtaining inaccurate estimates on future expected returns. 

Additionally, with the exponential increase in volumes of data, stock market analysts and particpants currently rely on non-AI methods (such as technical and fundamental analysis) for stock selection which are often arbitrary due to their limited ability to analyze raw data.

## Data 
10 year monthly stock data (Open, Close, High, Low) from May 2016 to April 2026 for stocks listed on the Pakistan Stock Exchange (PSX). 

## Methodology
1. **_Data Cleaning and Preprocessing_** 
    - This entails ensuring the stock data has full price information for the 10 year period and removal of null or outlier values. 
    - An Augmented Dickey Fuller (ADF) test was applied to the stock returns to ensure time series stationarity. 
    - Data is then normalized within the range (0,1) and split into 80% for training and 20% for testing.
2. _**LSTM Model Construction**_ 
    - A Keras random search methodology was applied for both univariate and multivariate models to select the most optimal combination of LSTM hyperparameters. 
    - For the multivariate LSTM model, 167 features consisting of lagged versions of returns and technical indicators were generated. 
    - 25 features were selected as model inputs using Recursive Feature Elimination (RFE) via a Random Forest regressor.
3. **_Portfolio Construction_** 
    - 10 stocks with the highest return predictions were selected for portfolio optimization. Here, 30-day return forecasts from each model were fed into a mean-variance optimization model. Portfolio performance metrics generated by each model's preditions were then compared.

## v2-daily-data-framework: Modular src/ Structure
The core logic previously embedded in the `Codes/` notebooks has been refactored into a modular, reusable `src/` package:

- `src/seed_utils.py` – loads `config.yaml` and applies the single global `random_seed` consistently across NumPy, Python's `random`, TensorFlow/Keras, and scikit-learn (`random_state`).
- `src/data_loader.py` – CSV loading/cleaning, return calculation, and multi-stock portfolio data loading.
- `src/feature_engineering.py` – config-driven technical indicator pipeline: SMA, EMA, RSI, MACD, Stochastic Oscillator, ATR, Bollinger Bands, OBV, VWAP, and standard OHLCV lag features.
- `src/feature_selection.py` – RFE-based feature selection utilities.
- `src/models_baseline.py` – classical baseline models: ARIMA and XGBoost, plus shared RMSE/MAE evaluation utilities.
- `src/models_deep_learning.py` – deep learning models: the univariate/multivariate LSTM builders and training helper.
- `src/portfolio_optimization.py` – Monte Carlo mean-variance portfolio simulation.

Configuration lives in `config.yaml` at the project root, and pinned dependency versions are listed in `requirements.txt`. The original notebooks in `Codes/` are retained for reference.

### Daily OHLCV Data Pipeline
`Data/` contains daily OHLCV (Open, High, Low, Close, Volume) history for the selected KSE-100 companies, spanning July 2016 - July 2026. `src/data_loader.py` was updated to support this:

- `load_stock_csv` now parses and retains the `Volume` column (including `K`/`M`/`B`-suffixed values) alongside OHLC.
- `chronological_train_val_test_split` performs a strict, non-randomized walk-forward split: 70% training / 15% validation / 15% testing, in date order.
- `scale_train_val_test` fits a `MinMaxScaler` on the training split only, then applies that same fitted scaler to the validation and test splits, preventing forward-looking data leakage.

### Feature Engineering Pipeline
`src/feature_engineering.py` generates trend/momentum indicators (SMA, EMA, RSI, MACD, Stochastic Oscillator), volatility/volume indicators (ATR, Bollinger Bands, OBV, VWAP), and standard lag features for the daily OHLCV data. Every indicator group and the lag-feature step can be toggled independently via the `feature_engineering` section of `config.yaml`, enabling future ablation experiments without code changes.
